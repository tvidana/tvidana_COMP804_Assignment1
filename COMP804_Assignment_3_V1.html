Name Web Attacks
DoS Attacks



Source: www.123rf.com/photo_65452190_victim-computer-laptop-with-target-lock-has-distributed-denial-of-service-ddos-attack-concept-design.html" /> 		A denial-of service (DoS) attack acts to prevent a system in responding to legitimate serivices requests. The victim network becomes flooded by illegitimate requests and it becomes impossible for the network to operate as it normally does and will result in a complete shutdown. Often, the network will have to come offline which leaves it vulnerable for other types of attacks as well

Learn more about DoS Attacks
Phishing Attacks



Source:https://acuityrm.com/blog/phishing-cyber-risk-a-practical-quantitative-approach/ 		A phishing attack occurs when a threat actor sends emails that seem legitimate, in the hope of retrieving sensitive information from victims. These emails will often have a fradulent link, which will ask the victim to enter their login credentials or other personal information, or even begin downloading malware, when the link is clicked. The threat actor is able to sit within the network undetected and go after other users in the same network

Learn more about Phishing Attacks
Ransomware



Source: https://www.reversinglabs.com/blog/the-week-in-security-ransomware-pytorch-supply-chain-attack 		Ransomware is when a victim network is held hostage until they agree to pay a ransom payment to the attacker. Once provided, the attacker promises to give the instructions on how to bring the fallen network back into operation. A network can become victim to ransomware via a fraud website or a malicious email attachment

Learn more about Ransomware Attacks
SQL Injection Attacks



Source: https://medium.com/@mycountryakash/sqli-injection-attack-fe34a64d1144 		Sructured Query Language (SQL) Injection acts to take advantage of networks that deend on databses to conduct daily business functions. This attack uses an SQL query sent from the client to a database on the server. Once the command is received, it is injected into a data plane in place of what is normally used, such as a password/login. The network can be penetrated once the server runs the command.If successful, an attacker is able to access sensitive data and modify/delete it, as well as gain adminstrator access and enforce a shutdown of the entire network.

Learn more about SQL Injection Attacks
Man-In-The-Middle Attacks



Source: https://www.malwarebytes.com/blog/news/2018/07/when-three-isnt-a-crowd-man-in-the-middle-mitm-attacks-explained 		Man-in-the-middle (MITM) attacks describe those attacks which make it possible for an attacker to access the data being sent back and forth between two computers or networks. While the attacker is spying on the data being exchanged between two parties, they are unaware that the attacker may be modifying or manipulating data before it reaches its destination. Without the use of encryption protocols or a VPN, networks can be victim to this.

Learn more about Man-In-The-Middle Attacks
Cryptojacking



Source: https://blog.knowbe4.com/cryptojacking-101-a-first-look-at-cryptomining-attacks 		Cryptojacking refers to when threat actors mine of crptocurrency using another network through unauthorised access. Attackers can gain access by encouraging a victim to access to fradulent link and residing in the network. Victims are often unaware of that crpto mining codes are working in the background.

Learn more about Crptojacking Attacks
Trojan Horses



Source: https://www.freepik.com/premium-vector/computer-virus-trojan-horse_5742915.htm 		Trojan Horse Attacks utilise a malicious code that is masked behind a program that looks legitimate from the outside. When a network prompts the program to run, the malware within is used to open a backdoor into the network through which attackers can gain access. Often these attacks can be prevented by users not downloading or installing programs unless the source is verified and trusted.

Learn more about Trojan Horse Attacks
Supply Chain Attacks



Source: https://cyberriskleaders.com/supply-chain-cyber-attacks-expected-to-quadruple-by-end-2021/ 		Supply Chain Attacks target third party vendors who work with the victim and offer vital services to them. Attackers use malicious codes within prgrams which result in users becoming infiltrated. These attacks can have a domino effect and result in large amounts of unauthorised access to data and severe business interruption.

Learn more about Supply Chain Attacks
Internet-of-Things Attacks



Source: https://www.wallarm.com/what/iot-attack 		Another type of attack is one that targets an Internet of Things (IoT) network, as they generally lack strict security protocols. Attackers are able to exploit this weakness and gain unauthorised access into a system to steal, manipulate and delete sensitive data as they wish. They can also install malware within the network, which can allow attackers to gain access to further personal data of the company

Learn more about Internet-of-Things Attacks
Cross-Site Scripting Attacks



Source: https://www.securecoding.com/blog/xss-attacks/ 		Cross-Site Scripting is when an attacker sends malicious scripting using clickable content, which is sent to the victim's search browser. Once clicked, the script begins to run. As the victim is logged in as a legitimate user, it does not seem unordinary. For example, the altered script can result in attackers being able to manipulate online banking transactions.

Learn more about Cross-Site Scripting Attacks
